SQL Injection - Ejemplos y Mitigación

Ejemplo 1: Inyección SQL en la autenticación

Ingresa tus credenciales:

Descripción: Este ejemplo demuestra una vulnerabilidad en el proceso de autenticación donde las credenciales del usuario son utilizadas directamente en una consulta SQL sin sanitizar.
Impacto: Un atacante puede omitir la autenticación ingresando `' OR '1'='1` como nombre de usuario o contraseña.
Solución: Utilizar consultas preparadas o procedimientos almacenados para evitar la inyección de código SQL.

Ejemplo 2: Inyección SQL en una búsqueda

Busca un producto:

Descripción: Este ejemplo muestra cómo una búsqueda de productos es vulnerable a la inyección SQL cuando el término de búsqueda se utiliza directamente en una consulta SQL.
Impacto: Un atacante puede extraer información sensible ingresando `'; DROP TABLE products;--` como término de búsqueda.
Solución: Utilizar consultas preparadas o procedimientos almacenados para evitar la inyección de código SQL.

Ejemplo 3: Inyección SQL en la actualización de datos

Actualiza tu información de usuario:

Descripción: Este ejemplo muestra cómo la actualización de información del usuario puede ser vulnerable a la inyección SQL cuando los nuevos datos se utilizan directamente en una consulta SQL.
Impacto: Un atacante puede modificar datos arbitrarios en la base de datos ingresando `'; UPDATE users SET role='admin';--` como nuevo nombre de usuario.
Solución: Utilizar consultas preparadas o procedimientos almacenados para evitar la inyección de código SQL.

Medidas de Mitigación

  • Utilizar consultas preparadas (prepared statements) para todas las interacciones con la base de datos.
  • Validar y sanitizar la entrada del usuario antes de utilizarla en consultas SQL.
  • Utilizar procedimientos almacenados (stored procedures) para encapsular las consultas SQL.
  • Restringir los privilegios de las cuentas de la base de datos utilizadas por la aplicación web.
